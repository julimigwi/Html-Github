Git Essentials

Git is a distributed version control system that allows multiple developers to work on a project simultaneously without interfering with each other's changes.

Key Concepts

  • Repository: A directory that contains all the files and their history.
  • Commit: A snapshot of the project at a specific point in time.
  • Branch: A separate line of development in a repository.
  • Merge: Combining changes from different branches.
  • Remote: A version of the repository hosted on a server.
  • Clone: Creating a local copy of a remote repository.
  • Push: Sending local changes to a remote repository.
  • Pull: Fetching changes from a remote repository and merging them into the local repository.
  • Staging Area: A place where changes are prepared before committing.
  • HEAD: A reference to the current commit in the repository.
  • Conflict: Occurs when changes in different branches cannot be automatically merged.
  • Tag: A marker for a specific commit, often used for releases.
  • Fork: A personal copy of someone else's repository, allowing for independent development.
  • Pull Request: A request to merge changes from one branch into another, often used in collaborative projects.
  • Cherry-pick: Applying a specific commit from one branch to another.
  • Rebase: Reapplying commits from one branch onto another, often used to maintain a clean project history.
  • Stash: Temporarily saving changes that are not ready to be committed.
  • Blame: Showing who made changes to each line of a file.
  • Revert: Creating a new commit that undoes changes made by a previous commit.
  • Submodule: A repository nested inside another repository, allowing for modular development.
  • Worktree: A separate working directory for a specific branch, allowing multiple branches to be worked on simultaneously.
  • Squash: Combining multiple commits into a single commit, often used to simplify commit history.
  • Bisect: A method for finding the commit that introduced a bug by performing a binary search through the commit history.
  • Hooks: Scripts that run at specific points in the Git workflow, allowing for automation of tasks like testing or deployment.
  • Config: Settings that control Git's behavior, such as user information and repository settings.
  • Alias: A shortcut for a longer Git command, allowing for easier usage.
  • Diff: A comparison of changes between commits, branches, or files, showing what has been added or removed.

Basic Commands

  • git init: Initialize a new Git repository.
  • git clone [url]: Clone a remote repository.
  • git add [file]: Stage changes for commit.
  • git commit -m "message": Commit staged changes with a message.
  • git status: Check the status of the repository.
  • git push: Push changes to the remote repository.
  • git pull: Fetch and merge changes from the remote repository.
  • git branch: List branches in the repository.
  • git checkout [branch]: Switch to a different branch.
  • git merge [branch]: Merge changes from another branch into the current branch.
  • git log: View the commit history.
  • git diff: Show changes between commits, branches, or files.
  • git reset [commit]: Reset the repository to a specific commit.
  • git revert [commit]: Create a new commit that undoes changes made by a previous commit.
  • git stash: Temporarily save changes that are not ready to be committed.
  • git tag [name]: Create a tag for a specific commit.
  • git remote add [name] [url]: Add a remote repository.
  • git remote -v: List remote repositories.
  • git fetch: Fetch changes from the remote repository without merging.
  • git cherry-pick [commit]: Apply a specific commit from one branch to another.
  • git rebase [branch]: Reapply commits from one branch onto another.
  • git bisect start: Start a bisect session to find a bug.
